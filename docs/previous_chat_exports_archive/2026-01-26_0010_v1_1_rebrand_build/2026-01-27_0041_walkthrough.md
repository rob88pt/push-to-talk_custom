# Language Selector & OpenAI Transcription Enhancements

I have implemented a new language selection feature for transcription and improved the OpenAI transcription pipeline by adding language hints and fixing the glossary support.

## Changes Made

### 1. Application Core & Configuration
- **Added `language` field** to `PushToTalkConfig` in [push_to_talk.py](file:///d:/Push_to_talk/src/push_to_talk.py).
- **Updated `PushToTalkApp`** to pass the selected language (or `None` for auto-detect) to the transcription service.

### 2. OpenAI Transcription Improvements
- **Language Hints in Prompt**: In [transcription_openai.py](file:///d:/Push_to_talk/src/transcription_openai.py), I now use the `prompt` parameter to provide a language hint (e.g., `Transcription language: es`) when a specific language is selected. This improves transcription consistency and performance for Whisper.
- **Glossary Support Fixed**: I fixed a bug where the `custom_glossary` was not being passed to OpenAI's transcription API. It is now correctly appended to the `prompt`.

### 3. GUI Enhancements
- **New Language Dropdown**: Added a "Transcription Language" combobox to the [api_section.py](file:///d:/Push_to_talk/src/gui/api_section.py) under Speech-to-Text settings.
- **Dynamic Wiring**: Updated [configuration_window.py](file:///d:/Push_to_talk/src/gui/configuration_window.py) to handle the new variable state and ensure it persists correctly.
